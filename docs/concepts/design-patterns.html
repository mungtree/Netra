Design patterns

Mini ChatUR leans on a small set of classic patterns. Learn them once and the whole codebase reads predictably.

Repository

Persistence hides behind trait interfaces. The engine asks a JobRepo to store a job; it neither knows nor cares that the implementation is SQLite.

  • Interfaces: ProjectRepo, JobRepo, BatchRepo, TemplateRepo.
  • Implementations: SqliteJobRepo and siblings in chatur-store.

Strategy

The batch reduce step is interchangeable. An Aggregator is a strategy; the batch picks one by id.

  • Interface: Aggregator.
  • Implementations: ConcatAggregator, SchemaMergeAggregator, held in an AggregatorRegistry.

Factory + Object Pool

Agent processes are expensive and must be capped. A factory creates transports; a pool bounds how many run at once, globally and per project.

  • Factory: TransportFactory → PiTransportFactory (real) / MockTransportFactory (tests).
  • Pool: AgentPool; a checkout is an AgentLease that releases its permits on drop.

Observer

Two observer channels carry events away from the runner. Producers do not know who is listening.

  • OutputSink (interface) — per-job event consumers; FileLogSink writes log files.
  • EventBus (interface) — broadcast pub/sub; BroadcastEventBus wraps a tokio broadcast channel.

Facade

One struct hides the whole subsystem. Callers (CLI, Tauri) touch only it.

  • Chatur wires store + agent pool + engine and exposes one method per operation.

Builder

Batches have many optional parts, so they are assembled fluently.

  • BatchBuilder — .prompt().target_project().strategy().build().

Command

A Job is a reified request: it carries everything needed to run later, can sit in a queue, be cancelled, retried, and persisted.

  • Job flowing through JobQueue → Scheduler → JobRunner.

Adapter

The pi RPC protocol is foreign and version-tied. An adapter layer translates it into the project's own vocabulary.

  • protocol.rs decodes pi's wire JSON; map_event() turns pi events into the normalized AgentEvent.

Dependency Injection

Anything non-deterministic or external is passed in, not hard-coded — so tests can substitute it.

  • Clock and IdGenerator — SystemClock / UuidGenerator in production, fakes in tests.
  • Every repo, transport, sink, and bus is injected as a trait object.
✓ Rule of thumb: if you see a trait in chatur-core, it is one of these patterns. Find its implementations to see the concrete behaviour.
