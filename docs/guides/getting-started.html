Getting started

Build and run Mini ChatUR — the headless CLI first (fewest dependencies), then the desktop shell.

Prerequisites

ForYou need
The library + CLI Rust 1.93+ (the workspace is edition 2024).
Running actual jobs pi v0.74.2 on PATH, and a reachable model server.
The desktop shell Also: Tauri system libs (webkit2gtk etc.), the Tauri CLI, and Node 20+. See the Tauri guide.
ⓘ Project, batch, and job listing commands work fully offline. Only running a job launches a pi process and therefore needs the model server up.

Build the workspace

# from the repository root
cargo build --workspace            # the 6 library crates
cargo test  --workspace            # run the test suite (uses fakes, no pi needed)

src-tauri is a separate Cargo workspace and is not built by --workspace — that is deliberate, so the core build needs no system webkit libraries.

Run the CLI

cargo build -p chatur-cli --release    # binary: target/release/chatur

# register a project, then ask the agent something
target/release/chatur project add myrepo /path/to/code
target/release/chatur run <project-id> summarize the architecture

Full command reference: CLI guide.

Run the desktop app

cd ui && npm install && cd ..
cd src-tauri && cargo tauri dev

cargo tauri commands must run from inside src-tauri/. Full instructions: Tauri guide.

Configuration

Both the CLI and the desktop app read chatur.toml from the current directory. Every field has a default, so the file is optional.

pi_binary = "pi"
data_dir  = ".chatur/data"     # SQLite database lives here
log_dir   = ".chatur/logs"     # per-job log files live here

[concurrency]
global_max      = 4            # max agent jobs at once, all projects
per_project_max = 2            # max agent jobs at once for one project

[default_model]
provider = "llamacpp"
model    = "qwen3.6-35b-a3b"

Config loading is handled by ChaturConfig.

The local model setup

The development setup runs models locally: provider llamacpp, model qwen3.6-35b-a3b, served by a llama.cpp OpenAI-compatible server at http://127.0.0.1:8888/v1, configured in ~/.pi/agent/config.json and models.json. The model server must be running before any job will complete.
