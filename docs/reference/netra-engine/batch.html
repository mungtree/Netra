netra-engine / batch.rs

The BatchExecutor — runs a batch's map step, waits, then runs its reduce step.

Path: crates/netra-engine/src/batch.rs

In plain terms

A batch is map/reduce. The executor's map turns every batch item into a job and drops it on the shared queue — so the scheduler runs those jobs normally, under the global cap. The executor then just waits for them and runs the reduce step on whatever succeeded.

Constant

REVIEWER_STRATEGY = "reviewer" (batch.rs:28) — the strategy id that triggers the agent-backed reduce path.

batch.rs:119 async fn execute(&self, batch: &Batch) -> Result<AggregatedResult>

The map-then-reduce body. Map: for each BatchItem — resolve the prompt, create a Job (tagged with batch_id), record its id back on the item, enqueue it, publish JobQueued. Then: collect() the outputs; error if there are no items or every job failed. Reduce: delegate to reduce().

Builds one item's final prompt: find the named BatchPrompt; load its body (inline, or fetched via TemplateRepo::get); substitute {{variables}} with render(); if the batch has an output_schema, append a "respond ONLY with JSON matching this schema" instruction.

batch.rs:183 async fn collect(&self, job_ids: &[JobId]) -> Result<Vec<AgentOutput>>

Polls JobRepo::get every poll_interval until each job is terminal, gathering the outputs of the ones that Completed. Failed/cancelled jobs are skipped, not fatal.

batch.rs:201 async fn reduce(&self, batch: &Batch, outputs: Vec<AgentOutput>) -> Result<AggregatedResult>

If the strategy is "reviewer", delegates to run_reviewer(); otherwise looks the strategy up in the AggregatorRegistry and calls aggregate(). An unknown strategy is CoreError::Invalid.

batch.rs:216 async fn run_reviewer(&self, batch: &Batch, outputs: Vec<AgentOutput>) -> Result<AggregatedResult>

The agent-backed reduce step. Joins every map output into one prompt asking a reviewer agent to consolidate/dedupe/rank, enqueues that as one standalone job (not a batch member), waits for it, and wraps its output as the AggregatedResult. Token usage sums the map outputs plus the reviewer call.

Private helper

batch.rs:274 fn render(body: &str, vars: &HashMap<String, String>) -> String

Replaces every {{name}} placeholder with its value; unknown placeholders are left untouched.

⚡ Ripple effects.
  • Shared queue is mandatory. The map step's queue.enqueue only results in execution because that queue is the one Scheduler drains. Hand the executor a different queue and batch jobs hang forever.
  • collect() couples this file to JobStatus::is_terminal — it relies on the runner setting a terminal status.
  • Driven by Netra::run_batch, which spawns run() on a background task.
  • resolve_prompt's schema instruction is what makes schema_merge viable downstream.
