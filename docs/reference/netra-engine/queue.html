InMemoryJobQueue — the FIFO list of pending jobs, shared by the API layer and the scheduler.

Path: crates/netra-engine/src/queue.rs

In plain terms

Jobs wait here until the scheduler is ready for them. It is a plain first-in-first-out list, but with two twists: it can be cloned cheaply and all clones share one queue (so the API and the scheduler each hold one), and it can be awaited — a scheduler can block on "tell me when a job arrives" instead of busy-polling.

⚡ Ripple effects.
  • The "clones share state" property is load-bearing: Netra hands the same queue to both queue_job and the Scheduler, and to the BatchExecutor. If a copy were made instead, batch jobs would never run.
  • enqueue's notify_one() is what wakes a parked scheduler — the link between enqueue and wait_for_job.
  • This queue is in-memory only — pending jobs do not survive a restart (though they are also persisted as Queued rows by JobRepo).
