netra-engine / bus.rs

BroadcastEventBus — the EventBus implementation, a fan-out over a tokio broadcast channel.

Path: crates/netra-engine/src/bus.rs

In plain terms

The engine needs to announce events to anyone listening — in practice, the desktop UI. This is the implementation of the EventBus trait. It wraps a tokio broadcast channel: every subscriber gets every event, independently.

bus.rs:48 fn subscribe(&self) -> BoxStream<'static, DomainEvent>

Returns an independent event stream. A slow subscriber that overruns the channel capacity silently skips the events it missed (Lagged errors are filtered out) rather than stalling publishers.

⚠ The "drop on lag" behaviour is why the channel capacity matters. During a multi-agent token burst, every AssistantText delta is a JobProgress event; too small a capacity and a slow UI loses output. The Tauri shell raises the capacity well above the default for exactly this reason.
⚡ Ripple effects.
  • publish is called by JobRunner (job events, including per-token JobProgress) and BatchExecutor (batch events).
  • subscribe feeds Netra::subscribe_events, which the Tauri shell forwards to the front-end.
