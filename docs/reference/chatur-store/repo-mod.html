chatur-store / repo/mod.rs

The repo module root: declares the four repositories and holds two helpers every one of them uses.

Path: crates/chatur-store/src/repo/mod.rs

In plain terms

All four SQLite repos do the same two boring things constantly: turn an sqlx error into a CoreError, and decode a JSON data column into a domain struct. Rather than repeat that code, it lives here once.

Functions

repo/mod.rs:23 pub(crate) fn store_err(error: sqlx::Error) -> CoreError

Maps any sqlx error to CoreError::Storage. Every repo call ends with .map_err(store_err).

repo/mod.rs:28 pub(crate) fn decode_data<T: DeserializeOwned>(row: &SqliteRow) -> Result<T>

Reads the data column off a row and deserializes it into T. This is the read half of the JSON-blob storage model.

Re-exports

The four repo types — SqliteBatchRepo, SqliteJobRepo, SqliteProjectRepo, SqliteTemplateRepo — from their submodules.

⚡ Ripple effects. decode_data is the single point where the stored JSON is trusted to match the current domain struct. Because storage is a JSON blob, a renamed or removed field can deserialize wrong at runtime without any compile error — this helper is where that would surface.
