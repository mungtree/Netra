chatur-store crate

Persistence and logging: SQLite-backed implementations of the repository traits, plus the file log sink.

What this crate is for

Everything that must survive a restart lives here. It implements the four repository traits over SQLite (via sqlx) and the OutputSink trait as a log-file writer. The rest of the workspace depends on the traits, never on SQLite directly.

Depends on chatur-core only.

The storage model

ⓘ Every entity is stored as one JSON data blob — the source of truth — alongside a few promoted columns used only for indexing and foreign keys. Consequence: a domain struct can gain a field with no schema migration; only changes that affect querying or relations need one.
