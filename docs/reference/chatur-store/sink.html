FileLogSink — writes every job's event stream to a dated log file.

Path: crates/chatur-store/src/sink.rs

In plain terms

This is the implementation of the OutputSink trait. As a job runs, every event is appended — as one JSON line — to <root>/<YYYY-MM-DD>/<job-id>.jsonl. The file is flushed after every single write, so the log survives even if the process crashes mid-job.

⚡ Ripple effects.
  • JobRunner holds a FileLogSink in its sinks list: dispatch() calls on_event per event, attempt() calls flush at the end.
  • Flush-per-event trades throughput for crash-safety — a job that emits many tokens does many fsync-class writes.
  • These .jsonl files are what the CLI guide tells you to inspect with jq.
