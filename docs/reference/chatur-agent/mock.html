chatur-agent / mock.rs

In-process test doubles — a fake transport and a fake factory — so tests run without spawning pi.

Path: crates/chatur-agent/src/mock.rs

In plain terms

Spawning a real AI agent in a unit test is slow and needs a model server. MockTransport instead replays a fixed list of events. This module is deliberately pub (not test-gated) so chatur-engine, chatur-store, and chatur-api can use it in their tests too.

Trait impls

MockTransport implements AgentTransport (mock.rs:71): send_prompt records the message and returns the script as a stream; abort/shutdown bump their counters. MockTransportFactory implements TransportFactory (mock.rs:106).

⚡ Ripple effects. If you add a method to the AgentTransport or TransportFactory trait, you must implement it here too — otherwise every downstream test stops compiling.
