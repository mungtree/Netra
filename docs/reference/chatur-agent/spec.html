chatur-agent / spec.rs

The AgentSpec — everything needed to launch one pi process, and the logic that turns it into command-line flags.

Path: crates/chatur-agent/src/spec.rs

In plain terms

Before you can spawn pi, you need to decide: which binary, which folder, which model, how many tools, resume a session or not. An AgentSpec bundles those choices. Its build_args() method is the single place where those choices become actual pi CLI arguments.

⚡ Ripple effects.
  • build_args() is called by RpcTransport::spawn — it is the bridge between the typed ToolPolicy/ModelRef and the real CLI. Change a ToolPolicy variant and this match must change.
  • AgentSpec instances are produced by ProjectSpecResolver::resolve, one per job.
