netra-api crate

The single entry-point library for embedders: one Netra facade that wires together the store, agent pool, and engine.

What this crate is for

Everything that must talk to NETRA starts here. The Netra facade owns a Database, an AgentPool, an InMemoryJobQueue, a BroadcastEventBus, a Scheduler, and a BatchExecutor, then exposes a clean async API — start, queue_job, create_batch, shutdown, and more. The crate depends on netra-store, netra-agent, and netra-engine but has no dependency on Tauri, so it is fully usable as a standalone library.

Depends on netra-core, netra-store, netra-agent, netra-engine.
