The OutputSink interface — an observer of one job's event stream.

Path: crates/chatur-core/src/traits/sink.rs

In plain terms

While a job runs, every event it produces is also handed to each registered sink. A sink decides what to do with events — today, write them to a log file; tomorrow, perhaps push them to a webhook or a metrics system.

Where is the difference from EventBus? The bus is one broadcast channel for high-level domain events; a sink is a per-job consumer of the raw agent event stream.

Implementations

  • FileLogSink (id "file-log") — writes each job's events as JSON Lines to a dated log file.
⚡ Ripple effects.
  • JobRunner holds a Vec<Arc<dyn OutputSink>>; its dispatch() calls on_event for every sink on every event, and attempt() calls flush when the job ends.
  • A slow or failing sink slows the job — on_event is awaited inline in the event loop.
