chatur-core / traits/mod.rs

The traits module root — gathers the eight interface files.

Path: crates/chatur-core/src/traits/mod.rs

In plain terms

This is the most architecturally important folder in the whole project. Every cross-crate dependency goes through one of these traits. The file itself just declares the eight submodules privately and re-exports them flat, so callers write chatur_core::traits::JobRepo.

The eight interface files

FileTrait(s)Implemented in
transport.rsAgentTransportchatur-agent
session.rsAgentSessionchatur-agent
repo.rsProjectRepo, JobRepo, BatchRepo, TemplateRepochatur-store
queue.rsJobQueuechatur-engine
bus.rsEventBus (+ DomainEvent)chatur-engine
sink.rsOutputSinkchatur-store
aggregator.rsAggregatorchatur-engine
support.rsClock, IdGeneratorchatur-core (impls included)
ⓘ Because these are interfaces, the trick to reading any of them is "find the implementations". Each trait page lists where its implementations live.
⚡ Ripple effects. Changing any trait signature here is a workspace-wide breaking change — every implementor and caller must be updated. This is by design: the trait is the contract.
