chatur-core / error.rs

The single error type returned across every chatur-core interface.

Path: crates/chatur-core/src/error.rs

In plain terms

Lots of different things can go wrong — a database query, a process crash, bad JSON. Rather than expose a dozen unrelated error types, the whole workspace funnels failures into one enum, CoreError. Higher crates translate their concrete failures into one of its variants, so callers only ever match on a stable, small set of cases.

error.rs:45 pub type Result<T> = std::result::Result<T, CoreError>;

Convenience alias used as the return type of nearly every function in the workspace.

⚡ Ripple effects.
  • The Transport variant is special: it is the only error RetryPolicy::is_retryable treats as retryable. If you reclassify a failure as Transport, the engine will start retrying it — and vice versa.
  • #[from] serde_json::Error means any ? on a serde call auto-converts into CoreError::Serde.
  • Adding a variant forces every exhaustive match on CoreError to be updated.
