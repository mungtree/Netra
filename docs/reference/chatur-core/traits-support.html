chatur-core / traits/support.rs

Injectable support services — time and identity — so tests can control them.

Path: crates/chatur-core/src/traits/support.rs

In plain terms

"What time is it?" and "give me a new UUID" both produce different answers every call — which makes tests non-deterministic. By hiding them behind traits, a test can inject a fake clock that always returns the same instant. This file ships both the traits and their production implementations.

Implements Clock by returning Utc::now(). Zero-sized; Copy + Default.

support.rs:30 pub struct UuidGenerator

Implements IdGenerator by returning Uuid::new_v4(). Zero-sized; Copy + Default.

⚡ Ripple effects. These are the project's clearest example of dependency injection. Note that several model constructors (Job::new, Batch::new, JobId::new) currently call Utc::now() / Uuid::new_v4() directly rather than through these traits — so the injection seam exists but is not yet threaded everywhere. New code that needs time or ids should accept a Clock / IdGenerator.
