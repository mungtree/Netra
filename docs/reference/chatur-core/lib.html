chatur-core / lib.rs

The crate root. Declares the four public modules and re-exports the most-used names.

Path: crates/chatur-core/src/lib.rs

In plain terms

This file is just a table of contents. It says "this crate contains four modules" and lifts CoreError and Result to the crate root so callers can write chatur_core::Result instead of chatur_core::error::Result. No logic.

Modules

  • error — the workspace error type.
  • ids — typed identifiers.
  • model — domain data types.
  • traits — interface traits.

Re-exports

pub use error::{CoreError, Result};

Makes the error type and result alias available directly as chatur_core::CoreError / chatur_core::Result.

⚡ Ripple effects. Adding a module here exposes it crate-wide. This file itself is rarely a source of bugs; the things it re-exports are. If you add a new domain type or trait, declare its module here.
