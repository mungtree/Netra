The Aggregator interface — the batch reduce-step strategy.

Path: crates/chatur-core/src/traits/aggregator.rs

In plain terms

A batch produces many answers. An aggregator is the rule for boiling them down to one. Different rules (concatenate, merge JSON, have a reviewer agent consolidate) are interchangeable — that is the Strategy pattern. Each strategy has an id string; a batch picks one by id.

Implementations

  • ConcatAggregator ("concat") — verbatim concatenation.
  • SchemaMergeAggregator ("schema_merge") — merges structured JSON, dedupes.
ⓘ The "reviewer" strategy is not an Aggregator — it needs to run another agent, so it is handled directly by BatchExecutor::run_reviewer.
⚡ Ripple effects. Implementations are held in the AggregatorRegistry keyed by id(). To add a strategy: implement this trait and register the instance — see the recipe on the maintenance prompts page.
