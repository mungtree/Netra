The Job — one prompt run against one project. The smallest unit of work in the system.

Path: crates/chatur-core/src/model/job.rs

In plain terms

A job is a reified request (the Command pattern): it carries everything needed to run a prompt later, can sit in a queue, be cancelled or retried, and be saved to disk. It starts Queued and ends in one of three terminal states.

Returns true for Completed, Failed, Cancelled — the states from which no further transition occurs.

⚡ Ripple effects.
  • JobStatus::is_terminal is the poll-exit condition for Chatur::wait_for_job and the BatchExecutor::collect loop — both spin until a job reports terminal.
  • The JobRunner owns every status transition and persists it through JobRepo::update.
  • Job is stored as a JSON blob; its status and project_id are also promoted to indexed SQL columns (see the schema).
