netra-agent / pool.rs

The AgentPool — caps how many pi processes run at once — plus the transport factory and lease types.

Path: crates/netra-agent/src/pool.rs

In plain terms

Agent processes are heavy; you cannot run unlimited ones. The pool hands out a limited number of "leases". acquire() blocks until both a global slot and a per-project slot are free, then spawns a transport. When the lease is dropped, the slots free up automatically. This is the Factory + Object Pool pattern.

Best-effort async shutdown: spawns a task on the current tokio runtime to call transport.shutdown(). The semaphore permits are freed synchronously regardless.

⚡ Ripple effects.
  • The global_max / per_project_max limits come straight from ConcurrencyConfig. Set them to 1 and the whole system runs strictly sequentially.
  • JobRunner::attempt calls acquire for every job — this is the real throttle on how hard the machine works.
  • P1 design note: a fresh transport is created per acquire and torn down on release; transport reuse is a planned later optimization.
