PiSession — wraps a transport and turns its event stream into one finished AgentOutput.

Path: crates/netra-agent/src/session.rs

In plain terms

The transport gives you a firehose of events. Most callers just want "the answer". PiSession::run drains that firehose: it appends every assistant-text fragment, sums usage, and watches for an error event, then hands back a single AgentOutput.

session.rs:38 async fn run(&self, prompt: &str) -> Result<AgentOutput>

Sends the prompt, then loops the event stream: AssistantText deltas are concatenated into the output text; Usage events are summed; an Error event is remembered and, after the stream ends, surfaced as CoreError::Agent. Thinking, tool, and turn-marker events do not affect the output. Finally it opportunistically parses the text as JSON — success fills structured, failure just means "plain text".
