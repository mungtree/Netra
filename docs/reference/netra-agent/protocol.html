netra-agent / protocol.rs

The pi RPC wire format: encoding requests, decoding and classifying every line pi emits.

Path: crates/netra-agent/src/protocol.rs · module is crate-private

In plain terms

This file is the translator (the Adapter pattern). One direction: turn a Rust request into a JSON line for pi's stdin. Other direction: take a JSON line from pi's stdout and decide what it is — an acknowledgement, an event, the end of a turn, or noise. It is deliberately defensive: unknown messages are ignored, never errors, because pi's protocol is version-tied.

See the pi RPC protocol concept page for the full picture.

⚡ Ripple effects.
  • parse_line is called for every stdout line by RpcTransport's read_loop.
  • If pi ships a protocol change, this is the file to update — and nowhere else, by design. A new AgentEvent variant in core only does something once map_event produces it.
  • The "ignore unknown" rule means a forgotten mapping fails silently (the event just never appears) rather than crashing a run.
