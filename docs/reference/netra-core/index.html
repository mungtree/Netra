netra-core crate

The foundation crate: domain data types and the trait interfaces every other crate implements. Zero I/O, no async runtime, no internal dependencies.

What this crate is for

Read this crate first. It defines what the system is made of (projects, jobs, batches, events) and the contracts the rest of the workspace fulfils. It spawns no processes, touches no database, and depends on no other internal crate — so it can be understood with no other context.

Everything here is one of two things: a plain data type (a struct/enum with construction helpers and no real behaviour) or a trait (an interface implemented in netra-agent, netra-store, or netra-engine).

Who depends on it

Everyone. netra-agent, netra-store, netra-engine, netra-api, and the Tauri shell all build on these types and traits. Changing a type or trait signature here ripples through the entire workspace — see the Cross-module map.
