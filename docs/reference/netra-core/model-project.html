netra-core / model/project.rs

The Project — a registered local code folder — plus how its model and tool permissions are described.

Path: crates/netra-core/src/model/project.rs

In plain terms

Before the agent can do anything, you register a project: a name and a folder. The project also carries defaults — which model to use and how much the agent is allowed to touch — that individual jobs inherit unless they override them.

project.rs:50 pub enum ToolPolicy { ReadOnly, Allowlist(Vec<String>), Full }

Controls which pi tools the agent may invoke. ReadOnly is the #[default] — analysis only, no mutations.

VariantMaps to pi flags
ReadOnly--no-tools (or a read-only set)
Allowlist(names)--tools <names>
Fullnothing — all tools enabled
⚡ Ripple effects.
  • ToolPolicy and ModelRef are translated into actual command-line flags by AgentSpec::build_args. Change a variant here and that mapping must change too.
  • default_model sits in the middle of the model-precedence chain resolved by ProjectSpecResolver: job override > project default > app default.
  • The UI's MainHeader renders tool_policy for the selected project.
