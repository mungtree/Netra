The job-queue interface — an ordered, mutable list of pending jobs.

Path: crates/netra-core/src/traits/queue.rs

In plain terms

Jobs wait their turn in a queue. The scheduler depends only on this trait, so the queue could be in-memory or persistent without the scheduler caring.

Implementations

  • InMemoryJobQueue — the only one; a shared FIFO with an async wait_for_job() helper.
⚡ Ripple effects.
  • enqueue is called by Netra::queue_job and by BatchExecutor (one call per mapped job).
  • dequeue is the heartbeat of the Scheduler loop.
  • cancel backs Netra::cancel_job when the job is still queued (not yet running).
