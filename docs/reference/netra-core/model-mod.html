netra-core / model/mod.rs

The model module root — gathers the six domain-type files and re-exports everything flat.

Path: crates/netra-core/src/model/mod.rs

In plain terms

The domain types are split across six small files for readability, but callers should not care which file a type lives in. This module declares the six submodules privately and then does pub use agent::*; (and so on) so that, for example, Job is reachable as netra_core::model::Job no matter which file defines it.

Submodules (private) & what they export

ModuleTypes
agentAgentEvent, TokenUsage, AgentOutput
aggregateAggregatedResult
batchBatch, BatchStatus, BatchPrompt, PromptSource, BatchTarget, AggregationSpec, BatchItem, BatchBuilder
jobJob, JobStatus
projectProject, ModelRef, ToolPolicy
templatePromptTemplate

These types are plain data carriers — no behaviour beyond constructors. Behaviour lives behind the trait interfaces.

⚡ Ripple effects. A flat pub use * means a name clash between two submodules would break the build. Keep type names unique across the model tree.
