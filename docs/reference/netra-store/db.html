netra-store / db.rs

The Database — opens the SQLite pool, runs migrations on connect, and vends repositories.

Path: crates/netra-store/src/db.rs

In plain terms

One Database = one open SQLite connection pool. Creating it also runs all migrations, so the schema is always current. It then acts as a factory: ask it for .jobs(), .projects() and so on to get repository handles — all sharing the same pool.

⚡ Ripple effects.
  • Migrations run on every connect — adding a migration file under migrations/ makes it apply automatically next launch.
  • Netra calls Database::connect at startup and holds the Database; all of its persistence flows through these repos.
  • in_memory() is what lets the test suite run with no file on disk.
