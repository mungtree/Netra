SqliteJobRepo — the JobRepo trait over the jobs table.

Path: crates/netra-store/src/repo/job.rs

In plain terms

Reads and writes Job rows. The full job is the JSON data column; project_id, batch_id, status and created_at are promoted columns so the engine can query "all running jobs" or "all jobs for this project" quickly.

⚡ Ripple effects.
  • update is the most-called write in the system — JobRunner hits it on every status transition, and BatchExecutor::collect polls get in a loop.
  • status_str must stay in sync with JobStatus's serde(rename_all = "snake_case") — if the promoted column and the blob disagree, list_by_status silently misses rows.
