chatur-api crate

The single entry-point library for embedders: one Chatur facade that wires together the store, agent pool, and engine.

What this crate is for

Everything that must talk to Mini ChatUR starts here. The Chatur facade owns a Database, an AgentPool, an InMemoryJobQueue, a BroadcastEventBus, a Scheduler, and a BatchExecutor, then exposes a clean async API — start, queue_job, create_batch, shutdown, and more. The crate depends on chatur-store, chatur-agent, and chatur-engine but has no dependency on Tauri, so it is fully usable as a standalone library.

Depends on chatur-core, chatur-store, chatur-agent, chatur-engine.
